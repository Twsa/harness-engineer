/>  <div class="inner-block-content rich-content">
    <p data-block-key="fb946">The world of AI is rapidly moving beyond single-purpose models towards intelligent, autonomous multi-agent systems. Building these multi-agent systems, however, presents new challenges. That is why today, we have <a href="https://cloud.google.com/blog/products/ai-machine-learning/build-and-manage-multi-system-agents-with-vertex-ai">introduced Agent Development Kit</a> (ADK) at <b>Google Cloud NEXT 2025</b>, a new open-source framework from Google designed to simplify the full stack end-to-end development of agents and multi-agent systems. ADK empowers developers like you to build production-ready agentic applications with greater flexibility and precise control.</p><p data-block-key="v2cv">ADK is the same framework powering agents within Google products like Agentspace and the Google Customer Engagement Suite (CES). By open-sourcing ADK, we aim to provide developers with powerful, flexible tools to build in the rapidly evolving agent landscape. The ADK is designed to be flexible, use different models and build production ready agents for different deployment environments.</p><h2 data-block-key="1ldl"><br/>Core Pillars of ADK: Build, Interact, Evaluate, Deploy</h2><p data-block-key="9l1p3">ADK provides capabilities across the entire agent development lifecycle:</p><p data-block-key="b0pbo"></p><ul><li data-block-key="253hh"><b>Multi-Agent by Design:</b> Build modular and scalable applications by composing multiple specialized agents in a hierarchy. Enable complex coordination and delegation.</li></ul><p data-block-key="cee1c"></p><ul><li data-block-key="1up9q"><b>Rich Model Ecosystem:</b> Choose the model that works best for your needs. ADK works with your model of choice – whether it is Gemini or your any model accessible via Vertex AI Model Garden. The framework also offers LiteLLM integration letting you choose from a wide selection of models from providers like Anthropic, Meta, Mistral AI, AI21 Labs, and many more!</li></ul><p data-block-key="1pv4e"></p><ul><li data-block-key="68g93"><b>Rich Tool Ecosystem:</b> Equip agents with diverse capabilities: use pre-built tools (Search, Code Exec), Model Context Protocol (MCP) tools, integrate 3rd-party libraries (LangChain, LlamaIndex), or even use other agents as tools (LangGraph, CrewAI, etc).</li></ul><p data-block-key="ced47"></p><ul><li data-block-key="53lls"><b>Built-in streaming:</b> Interact with your agents in human-like conversations with ADK&#x27;s unique bidirectional audio and video streaming capabilities. With just a few lines of code, you can create natural interactions that change how you work with agents – moving beyond text into rich, multimodal dialogue.</li></ul><p data-block-key="3gq14"></p><ul><li data-block-key="4d03o"><b>Flexible Orchestration:</b> Define workflows using workflow agents (<code>Sequential</code>, <code>Parallel</code>, <code>Loop</code>) for predictable pipelines, or leverage LLM-driven dynamic routing (<code>LlmAgent</code> transfer) for adaptive behavior.</li></ul><p data-block-key="fja8q"></p><ul><li data-block-key="8fm2"><b>Integrated Developer Experience:</b> Develop, test, and debug locally with a powerful CLI and a visual Web UI. Inspect events, state, and agent execution step-by-step.</li></ul><p data-block-key="8v7r8"></p><ul><li data-block-key="e0mq7"><b>Built-in Evaluation</b>: Systematically assess agent performance by evaluating both the final response quality and the step-by-step execution trajectory against predefined test cases.</li></ul><p data-block-key="4o0vu"></p><ul><li data-block-key="dnsck"><b>Easy Deployment</b>: Containerize and deploy your agents anywhere.</li></ul><h2 data-block-key="bnueo"><br/>Getting started with your first agent</h2><p data-block-key="4vlpb">While we encourage you to explore the examples in the <a href="https://google.github.io/adk-docs/">docs</a>, the core idea is Pythonic simplicity. You define your agent&#x27;s logic, the tools it can use, and how it should process information. ADK provides the structure to manage state, orchestrate tool calls, and interact with the underlying LLMs. Here is an illustrative example of a basic agent.</p><p data-block-key="8lgik">The code can be found in the <a href="https://google.github.io/adk-docs/agents/llm-agents/">quickstart guide</a>.</p>

The way you define your agent (the core logic within <code>agent.py</code>) is the same regardless of how you choose to interact with it.     <h2 data-block-key="fb946">Building Multi-Agent Applications with ADK</h2><p data-block-key="20j34">ADK truly shines when you move beyond single agents to build collaborative multi-agent systems that leverage tools. Imagine creating a team of specialized agents where a primary agent can delegate tasks based on the conversation. ADK makes this easy through hierarchical structures and intelligent routing.</p><p data-block-key="amh0c">Let&#x27;s walk through an illustrative example – a <code>WeatherAgent</code> that handles weather queries but delegates greetings to a specialized GreetingAgent.</p><p data-block-key="dh1o8"><b><br/>1. Define a Tool:</b> Agents use tools to perform actions. Here, our <code>WeatherAgent</code> needs a tool to fetch weather data. We define a Python function; ADK uses its <code>docstring</code> to understand when and how to use it.</p>
</div>  <div class="inner-block-content code-block line-numbers">

    mock_weather_db = {
        &quot;newyork&quot;: {&quot;status&quot;: &quot;success&quot;, &quot;report&quot;: &quot;The weather in New York is sunny with a temperature of 25°C.&quot;},

    <h3 data-block-key="fb946"><b>How Delegation Works:</b></h3><p data-block-key="98g0j"></p><ul><li data-block-key="6aop0">The default agent behavior is to allow delegation.</li></ul><p data-block-key="2rpg3"></p><ul><li data-block-key="411mr">When processing a user message, the LLM considers the query, the current agent&#x27;s <code>description</code>, and the <code>description</code> fields of related agents (parent / sub agents defined in the hierarchy).</li></ul><p data-block-key="2l6ch"></p><ul><li data-block-key="62n6s">If the LLM determines another agent is a better fit based on its description (e.g., user says &quot;Hi&quot;, matching the <code>GreetingAgent</code> description, it initiates a transfer.</li></ul><p data-block-key="f1i25"></p><p data-block-key="9k12">Clear, distinct descriptions are vital! The LLM uses them to route tasks effectively.</p><p data-block-key="8tp29">In this setup, if a user starts with &quot;Hi&quot;, the <code>WeatherAgent</code> (if it&#x27;s the root agent processing the input) can recognize it&#x27;s not a weather query, see the <code>GreetingAgent</code> is suitable via its description, and automatically transfer control. If the user asks &quot;What&#x27;s the weather in Chicago?&quot;, the <code>WeatherAgent</code> handles it directly using its <code>get_weather</code> tool.</p><p data-block-key="a4aju">This example demonstrates how ADK&#x27;s hierarchical structure and description-driven delegation allow you to build organized, maintainable, and sophisticated multi-agent applications.</p><h2 data-block-key="2850f"><br/>Completing the Lifecycle: Evaluation and Deployment</h2><p data-block-key="cu93j">Building intelligent agents like our weather agent is foundational, but bringing them reliably to users involves crucial next steps: rigorous evaluation and seamless deployment. Before going live, ensuring your agent behaves predictably and correctly is paramount. ADK&#x27;s integrated evaluation tools are designed precisely for this, letting you systematically test execution paths and response quality against predefined datasets, like <code>evaluation.test.json</code> or <code>test.json</code>. You can run these checks programmatically within your test suites using <code>AgentEvaluator.evaluate()</code>. You can also use evaluation directly via the ADK eval command-line tool or via the web UI.</p><p data-block-key="41uhc">Once you&#x27;re satisfied with performance, ADK offers a clear and streamlined path to production through the option to deploy to any container runtime or using its integration with Vertex AI Agent Engine. This allows you to leverage a fully managed, scalable, and enterprise-grade runtime, completing the development lifecycle and empowering you to move from sophisticated prototypes to robust, production-ready agentic applications.</p><h2 data-block-key="2ns13"><br/>Choosing the framework for you: ADK or Genkit?</h2><p data-block-key="3ja90">As you explore the possibilities of building multi-agent systems with ADK, you might be wondering how it fits into the broader landscape of GenAI development tools from Google. While a variety of SDKs and frameworks are available, such as the <a href="https://github.com/firebase/genkit">Genkit framework</a>, it&#x27;s helpful to understand ADK&#x27;s relative focus. Here&#x27;s a quick comparison:</p><h3 data-block-key="7lbs1"><b><br/>Agent Development Kit:</b></h3><p data-block-key="6ge1g"></p><ul><li data-block-key="cpuku">Optimized for complex agents and multi-agent systems, it provides higher-level abstractions for agent development with built-in integration for LiteLLM and Vertex AI Model Garden supporting a variety of models.</li></ul><p data-block-key="aqkod"></p><ul><li data-block-key="d5aat">Focuses on defining agent behaviors and interactions.</li></ul><p data-block-key="j0el"></p><ul><li data-block-key="14dmi">Supports bidirectional streaming.</li></ul><h3 data-block-key="8nqtc"><b><br/>Genkit:</b></h3><p data-block-key="4m9ol"></p><ul><li data-block-key="9mrp0">Provides fundamental building blocks for building a large variety of AI powered experiences.</li></ul><p data-block-key="892iu"></p><ul><li data-block-key="8c74p">Includes developer tooling for iterating, testing and debugging your AI related interactions.</li></ul><p data-block-key="dp3rm"></p><ul><li data-block-key="dj4b6">Support a wide variety of large language models from Google AI, Vertex AI, and from 3rd parties through community plugins.</li></ul><h3 data-block-key="eegsh"><b><br/>Choosing the Right Tool</b></h3><p data-block-key="fl2vk">Ultimately, the best choice depends on your project&#x27;s specific goals. If you are building intricate, collaborative agent systems within a well-defined framework, ADK offers a powerful solution. For many other GenAI projects requiring flexibility and broad model support, Genkit is an excellent choice.</p><h2 data-block-key="3t4ml"><br/>ADK works anywhere, but optimized for Google Cloud</h2><p data-block-key="b5281">While the Agent Development Kit (ADK) offers flexibility to work with various tools, it is optimized for seamless integration within the Google Cloud ecosystem, specifically with Gemini models and Vertex AI. This tailored design allows developers to fully leverage the advanced capabilities of Gemini, such as the enhanced reasoning and tool use found in Gemini 2.5 Pro Experimental, and provides a direct, native pathway to deploy these agents onto Vertex AI&#x27;s fully-managed, enterprise-grade runtime for scalability.</p><p data-block-key="2uh8">Crucially, this deep integration extends to your broader enterprise landscape; ADK enables agents to connect directly to systems and data through over 100 pre-built connectors, utilize workflows built with Application Integration, and access data stored in systems like AlloyDB, BigQuery, and NetApp without requiring data duplication.</p><p data-block-key="drn9c">Additionally, agents built with ADK can securely tap into your organization&#x27;s existing API investments managed through Apigee, further enhancing their capabilities by leveraging established interfaces.</p><p data-block-key="f756a">This comprehensive connectivity across advanced AI models, scalable deployment, diverse data sources, and existing APIs makes ADK exceptionally powerful when used within the Google Cloud environment.</p><h2 data-block-key="95li4"><br/>Build the next generation of Agents with ADK</h2><p data-block-key="9avrt">The Agent Development Kit (ADK) provides a powerful, flexible, and open-source foundation for building the next generation of AI applications. It tackles the core challenges of multi-agent development by offering:</p><p data-block-key="8ngh5"></p><ul><li data-block-key="dam5p"><b>Precise control</b> over agent behavior and orchestration.</li></ul><p data-block-key="9gfhv"></p><ul><li data-block-key="53p32">A <b>rich ecosystem</b> for tools and integrations.</li></ul><p data-block-key="215na"></p><ul><li data-block-key="8coiu">An <b>integrated developer experience</b> for building and debugging.</li></ul><p data-block-key="15sth"></p><ul><li data-block-key="ena9c">A <b>robust evaluation framework</b> essential for reliable agents.</li></ul><p data-block-key="719hk"></p><ul><li data-block-key="bkkjl">A <b>clear path to deployment</b>, including managed options.</li></ul><p data-block-key="delq5"></p><p data-block-key="5j7jk">We&#x27;re incredibly excited to see what you build with ADK!</p><p data-block-key="7qo4p"><b>Explore the Code:</b> <a href="https://google.github.io/adk-docs">Official ADK Documentation</a>.</p>
